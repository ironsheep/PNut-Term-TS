# Pure Process Documentation

## Purpose
This folder contains **portable methodologies** that can be copied to ANY project, regardless of technology stack, tools, or domain. These documents represent universal software engineering wisdom gained through experience.

## Intent
- **100% Portable**: No tool-specific commands or project-specific details
- **Universal Principles**: Applies to any software development effort
- **Experience-Based**: Refined through actual project execution
- **Continuously Improved**: Updated as new patterns emerge

## What Belongs Here
- Planning methodologies and workflows
- Sprint execution patterns
- Task generation and estimation approaches
- Quality assurance strategies
- Project lifecycle management
- Knowledge preservation techniques
- Team collaboration patterns

## What Does NOT Belong Here
- Any mention of specific tools (Claude, todo-mcp, etc.)
- Project-specific configurations or requirements
- Technology-specific patterns
- Company or product details

## Key Documents
- **PLANNING-METHODOLOGY.md**: How to plan projects effectively
- Future additions as patterns are discovered and refined

## Usage
Copy this entire folder to any new project to immediately benefit from accumulated process wisdom. Adapt examples to your specific context while preserving the underlying principles.
