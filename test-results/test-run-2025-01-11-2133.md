### Critical Issue: RTS Mode Switch Error - FIXED
When changing control line mode from DTR to RTS in preferences, a renderer error occurred.

**Root Cause**: Template literal interpolation issue in `updateControlLineUI()` method.
- The code had `'${mode}'` with single quotes around the template literal placeholder
- This prevented TypeScript from interpolating the `mode` variable
- The literal string `${mode}` was being passed to the renderer, causing a JavaScript syntax error

**Fix Applied**: Changed template literal interpolation in lines 1776-1777 and 1797:
- Before: `resetButton.textContent = '${mode}';`
- After: `resetButton.textContent = '` + mode + `';`
- Applied same fix to `dataset.line` assignments
